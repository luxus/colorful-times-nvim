COLORFUL-TIMES                                             *colorful-times*

Colorful Times is a fast and simple Neovim plugin that automatically sets
your colorscheme based on your system appearance (light/dark mode) or based
on a time-of-day schedule. It is highly optimized, ensuring zero impact on
Neovim startup times via asynchronous `vim.uv` module spawning.

1. Introduction                                        *colorful-times-introduction*

This plugin aims to seamlessly match your terminal or OS's dark/light
preference dynamically as well as manage daytime/nighttime colorschemes.

It achieves zero-blocking startup overhead by:
1. Returning early with fallback defaults during the initial require block.
2. Synchronously initializing only necessary states to avoid UI flash.
3. Checking actual system preferences using non-blocking asynchronous `uv.spawn`.

Requires Neovim >= 0.12.0. The TUI is built in and has no optional picker
dependency.

2. Configuration                                       *colorful-times-configuration*

To initialize and setup the plugin, use `require('colorful-times').setup(opts)`.

Default configuration:
>lua
    require("colorful-times").setup({
      enabled = true,
      refresh_time = 5000,           -- milliseconds between system appearance polls
      system_background_detection = nil, -- Optional command/function for Linux
      system_background_detection_script = nil, -- Optional custom script path
      persist = true,                -- set false to disable TUI state persistence

      default = {
        colorscheme = "default",
        background = "system",       -- "light", "dark", or "system"
        themes = {
          light = nil,
          dark = nil,
        },
      },
      schedule = {},
    })
<

Schedule syntax:
>lua
    schedule = {
      { start = "08:00", stop = "18:00", colorscheme = "day_theme", background = "light" },
      { start = "18:00", stop = "08:00", colorscheme = "night_theme", background = "dark" },
    }
<

`persist` ~
    When `true` (default), TUI changes are written to:
    `~/.local/share/nvim/colorful-times/state.json`
    and loaded at startup. Set to `false` to opt out.

Theme resolution order:
1. runtime session hold, when active
2. matching schedule entry
3. `default.background`
4. `default.colorscheme` or `default.themes.light` / `.dark`

If the resolved background is `system`, Colorful Times applies a safe fallback
first and then updates asynchronously once the actual light/dark system
background is detected.

`system_background_detection` ~
    Platform-independent override. Set to a function returning `"dark"` or
    `"light"`, or a command table like `{ "gsettings", "get", "..." }`.
    These overrides take priority over the built-in platform detection.

`system_background_detection_script` ~
    Linux only. Path to a custom shell script for detecting system
    background mode. The script should exit with code 0 for dark mode
    and code 1 for light mode. If the script is not found or not
    executable, an error is displayed and the fallback background is used.
    When unset (nil), KDE/GNOME auto-detection is used when available.

3. Commands                                            *colorful-times-commands*

                                                       *:ColorfulTimes*
:ColorfulTimes
    Open the interactive schedule manager TUI.

                                                       *:ColorfulTimesEnable*
:ColorfulTimesEnable
    Enable the plugin.

                                                       *:ColorfulTimesDisable*
:ColorfulTimesDisable
    Disable the plugin.

                                                       *:ColorfulTimesToggle*
:ColorfulTimesToggle
    Toggle the plugin enabled/disabled state.

                                                       *:ColorfulTimesReload*
:ColorfulTimesReload
    Reload the plugin configuration from disk (re-reads state.json).

                                                       *:ColorfulTimesStatus*
:ColorfulTimesStatus
    Show the current resolved colorscheme, background, state, and
    detection backend information. If a session hold is active, the status
    command output includes `held: true`; API status still reports
    `pinned: true` and `source: session_pin` for compatibility.

                                                       *:checkhealth*
:checkhealth colorful-times
    Run the health check to verify Neovim version, vim.uv availability,
    inline TUI availability, real state-directory writability, detection
    backend availability, and schedule validity.

4. API                                                 *colorful-times-api*

`require('colorful-times').setup(opts)`                *colorful-times.setup()*
    Initializes the plugin and applies the current theme. Merges persisted
    state from `state.json` on top of `opts` when `persist = true`.

`require('colorful-times').toggle()`                   *colorful-times.toggle()*
    Toggles the plugin enabled/disabled state.

`require('colorful-times').enable()`                   *colorful-times.enable()*
    Enables the plugin and persists that state when persistence is enabled.

`require('colorful-times').disable()`                  *colorful-times.disable()*
    Disables the plugin and persists that state when persistence is enabled.

`require('colorful-times').reload()`                   *colorful-times.reload()*
    Rebuilds the live configuration from the original setup config and then
    re-reads persisted state from `state.json` when `persist = true`.

`require('colorful-times').open()`                     *colorful-times.open()*
    Opens the schedule manager TUI. Equivalent to `:ColorfulTimes`.

`require('colorful-times').status()`                   *colorful-times.status()*
    Returns a table describing the currently resolved source, colorscheme,
    background, persistence state, session hold state, and detection backend.

`require('colorful-times').pin_session(cs, bg, resolved_bg)`
                                                        *colorful-times.pin_session()*
    Hold a colorscheme/background for the current Neovim session. The hold is
    runtime-only, overrides schedules and system detection, and is never
    persisted to `state.json`.

`require('colorful-times').unpin_session()`             *colorful-times.unpin_session()*
    Remove the active session hold and return to normal schedule resolution.

`require('colorful-times').session_pin()`               *colorful-times.session_pin()*
    Return the current runtime session hold table, or nil when no hold is active.

5. TUI — Schedule Manager                              *colorful-times-tui*

Run `:ColorfulTimes` to open a keyboard-driven floating window for managing
your time-based colorscheme schedule and default theme settings. The TUI uses
one floating window and one scratch buffer. Adding, editing, theme selection,
and background selection all happen inline in that same buffer.

Modes~
    browse          View status, defaults, timeline, and schedule rows.
    edit            Edit a schedule row in an inline drawer.
    theme_select    Filter and choose colorschemes inline with live preview.
    bg_select       Cycle system/light/dark in an inline segmented control.

Keymaps~
    Tab           Switch focus between Defaults and Schedule
    j / <Down>    Move selection down inside focused panel
    k / <Up>      Move selection up inside focused panel
    a             Add a new schedule entry
    e / <Enter>   Edit selected schedule row or default row
    d / x         Delete the selected schedule entry (with confirmation)
    H             Hold current theme for this session; press again to release
    t             Toggle the plugin enabled/disabled
    r             Reload configuration
    ?             Show keymap help
    q / <Esc>     Close the TUI

Edit mode keymaps~
    Tab / j / k    Move between fields
    0-9 / :        Replace the active start/stop time; `14` becomes `14:00`
    <Enter>        Open inline theme/background selector for active field
    S              Save the draft, apply it, and persist it
    O              Hold current draft for this session; press again to release
    <Esc>          Cancel and restore the preview snapshot

Theme selector keymaps~
    type text      Filter colorschemes
    <BS>           Delete one filter character
    j / k          Move selection and preview live
    <Enter>        Confirm the selected colorscheme
    <Esc>          Cancel and restore the previous preview

Background selector keymaps~
    h / l / j / k  Cycle system/light/dark and preview live
    <Enter>        Confirm the selected background
    <Esc>          Cancel and restore the previous preview

Preview lifecycle~
    Entering add/edit mode captures the current colorscheme and background.
    Moving through theme choices or cycling background applies a live preview.
    Saving commits the preview. Canceling restores the exact captured
    colorscheme/background.

Session hold~
    Press `H` to hold the current resolved theme for this Neovim session.
    In edit mode, `O` holds the current draft preview. While held,
    automatic schedule changes and system background changes do not apply a
    different theme. Press the same key again to release. The hold is
    runtime-only, not persisted, shown in the TUI header, and removed by
    restarting Neovim.

Add defaults~
    New schedule entries default to the current resolved colorscheme and
    background. Time defaults use the displayed chronological schedule view:
    start is the stop time of the last displayed entry, and stop is the start
    time of the first displayed entry. With no entries, the defaults are
    08:00-18:00.

The TUI also shows the theme resolution order inline:
`schedule entry > default.background > default theme override`

Changes made in the TUI are immediately applied and persisted to disk
(unless `persist = false`). Runtime preview snapshots and session holds are
never persisted.
