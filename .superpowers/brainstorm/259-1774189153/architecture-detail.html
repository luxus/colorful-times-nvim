Full Rewrite Architecture

Module map for the new colorful-times. Everything below the dashed line is lazy-loaded.

colorful-times — module structure
── Entry point (always loaded) ──────────────────────
lua/colorful-times/init.lua — tiny stub: config defaults, metatable lazy-loader, type annotations
── Lazy-loaded modules (only loaded on first use) ──
core.lua
State machine + timer orchestration
  • enable / disable / toggle
  • schedule next change timer
  • appearance poll timer
  • FocusLost/FocusGained pause
  • apply_colorscheme()
schedule.lua
Schedule parsing + matching
  • parse_time() / validate()
  • preprocess_schedule()
  • get_active_entry(time)
  • next_change_at(time)
  • overnight handling
system.lua
OS dark/light detection
  • get_background(cb, fallback)
  • macOS: uv.spawn defaults
  • Linux: KDE / GNOME / custom
  • cached sysname
  • pauses when unfocused
state.lua
JSON persistence layer
  • load() / save()
  • ~/.local/share/nvim/colorful-times/state.json
  • TUI overrides stored here
  • merged over init.lua config on load
  • vim.json encode/decode
tui.lua — requires snacks.nvim
Table Manager TUI + forms
Schedule Table
  • floating window
  • ↑↓ navigate rows
  • [a] add, [e] edit, [d] delete
  • [t] toggle, [r] reload, [q] quit
Add / Edit Form
  • Snacks.input for times
  • Snacks.picker for colorscheme
  • live preview on colorscheme select
  • vim.ui.select for background
Commands
  • :ColorfulTimes — open TUI
  • :ColorfulTimesToggle
  • :ColorfulTimesReload
  • Lua API unchanged
── Data flow ────────────────────────────────────────
setup(opts) → merge opts + state.json → preprocess_schedule → apply_colorscheme → start timers
timer fires → get_active_entry → apply_colorscheme → schedule next
TUI edit → validate → update config → state.save() → core.reload()
FocusLost → pause appearance poll timer (zero CPU when unfocused)
